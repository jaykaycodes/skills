# Review Scorecard

Score each category 0-4, then summarize top risks.

## Categories
- Usability
- Visual hierarchy
- Accessibility
- Responsive behavior
- Feedback and state design
- Performance
- Consistency/system fit

## Score meanings
- 0: Broken / severe regressions
- 1: Major issues block good experience
- 2: Functional with notable quality gaps
- 3: Strong with minor issues
- 4: Excellent and ship-ready

## Findings format
- `[Severity] Short issue title`
- **Where:** component/page
- **Impact:** user consequence
- **Fix direction:** concrete change

Severity:
- Critical
- Major
- Minor
- Polish

## Exit criteria
- No Critical issues
- Major issues either resolved or consciously accepted with rationale
- Accessibility and responsive risks explicitly addressed
